# Wiki Source Meta — Runtime Review Methodology (MVP)

## Summary
Canonical MVP methodology for review REPRODUCIBILITY — the same review inputs should yield the same verdicts run-to-run. Defines the three reproducibility layers (which items / which checks / which verdict), the leg-classified HARD-implies-FAIL / SOFT-implies-RISK verdict rubric, the ensemble protocol (k>=3, default 5, 3-of-5 majority, contested -> escalate to HUMAN), RISK-in-headline derivation, and the finding-trace convention. Framework-independent; create-runtime-review-checklist is the operational instance. Empirically validated via the CR-056 2x2 re-test.
