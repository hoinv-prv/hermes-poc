第1章 はじめに

「INTARFRM」およびMigration CJC for INTARFRM（注2）と連携して動作します。したがって、本機能を利用するためには、これらの製品
が必要です。
「INTARFRM」は、クラウド時代に対応し、要件定義から設計・開発、運用・保守に至る「ソフトウェアライフサイクル」全体を支えるアプリ
ケーションフレームワーク製品です。
注1：アプリケーション開発において、開発量の削減と標準化のために用いるもの
注2：INTARFRM を利用して、JavaクライアントとCOBOLプログラムとの連携システムを効率的に開発可能なサービス（CJC：COBOL
Java Coordinateの略）
参照
「INTARFRM」の詳細は、<https://www.fujitsu.com/jp/solutions/infrastructure/dynamic-infrastructure/afw/> を参照してください。
「 Migration CJC for INTARFRM 」 の 詳細 は 、 <https://www.fujitsu.com/jp/solutions/infrastructure/dynamic-infrastructure/sdas/

# 第2章 連携機能の概要

第2章 連携機能の概要
ここでは、INTARFRM 連携機能の概要を説明します。
COBOLプログラムでは、ディスプレイ装置に画面を表示し、表示した画面からデータを入力すること（以降、画面入出力といいます）が
できます。
画面入出力を行う機能の1つに、表示ファイル機能があります。表示ファイル機能は、様々に定義された画面定義体を使って画面入出力を
行うことができるため、従来から多くの業務を実現するために利用されてきました。
表示ファイル機能の画面入出力では、開発時に関連製品である「FORM」を利用して画面を定義し、運用時に「MeFt」や「MeFt/Web」な
どの製品を利用して画面との入出力処理を行います。
これらには、以下のメリットとデメリットがありました。
・ メリット
－ 従来から培われたノウハウの集積がある
－ 独自技術による表現力・操作性に長けている
・ デメリット
－ クライアントシステムにランタイムやプラグインの導入が必要
－ インターネット環境に非対応
－ 他システムへの移行の汎用性が低い
一方で、画面入出力のクライアントには、初期導入が容易なWebブラウザの利用がスタンダードになっています。この場合、Webブラウザ上
に画面を表示するためのWebアプリケーションには、以下の理由からJavaクライアントが利用されるのが一般的です。
・ オープンソースのプラットフォーム
・ プラットフォーム無依存
・ 豊富なライブラリが利用可能
INTARFRM 連携機能では、これら表示ファイル機能のデメリットを解決し、スタンダードなJavaクライアントを利用するCOBOLプログラムの
画面入出力を実現します。
また、INTARFRM 連携機能を利用することで、従来のCOBOLプログラムで利用している既存資産を有効に活用しつつ、スムーズな移行を
実現できます。
画面情報定義体からJava（Servlet/JSP）に変換（注1）
「FORM」で作成した画面定義体を入力して、以下のソースファイルを生成します。
－ Javaクライアントが使用する画面レイアウトやレコードを定義したServlet/JSP
－ COBOLプログラムが使用するレコードを定義したCOBOL登録集
スタンダードなJavaクライアントを利用
開発系ツールが生成するServlet/JSPと組み合わせることで、画面制御はJavaクライアントを利用できます。
手続き型プログラミングで作成したアプリケーションで画面入出力を実現
表示ファイル機能の特徴である手続き型プログラミングで作成したアプリケーションを移行できます。アプリケーションをクラス化したり、
CORBAアプリケーションへ変更したりする必要はありません。
COBOLプログラムのソースファイルをそのまま（注2）利用可能
COBOLプログラムのソースファイルのロジックや構成を変更する必要はありません。
注1：Migration CJC for INTARFRMの機能です。NetCOBOL製品では提供していません。
注2：COBOLプログラムのリビルドは必要です。画面定義体から取り込んでいたレコード定義を、開発系ツールが生成するCOBOL
登録集から取り込むように修正する必要があります。一部の非互換機能については、COBOLプログラムの修正が必要となります。
- 2 -INTARFRM 連携機能を利用することで、既存のCOBOL資産から高い互換性を維持したまま、Javaクライアントと連携したインターネット
環境に対応したシステムを構築できます。
INTARFRM 連携機能には、以下の機能があります。
［1］ 画面入出力機能
画面入出力機能とは、JavaクライアントからCOBOLプログラムを実行して、Web ブラウザ上で画面入出力できる機能です。
画面入出力機能では、これまで表示ファイル機能で利用してきた既存のCOBOLプログラムを活用して、簡単にJavaクライアントと連携し
たインターネット環境に移行することができます。
INTARFRM 連携機能で起動できるCOBOLプログラムは、プロセスモードだけです。また、DLL（ダイナミックリンクライブラリ）のCOBOLプ
ログラムを実行することはできません。
プロセスモードについて
管理サービスはJavaクライアントから要求を受信すると、新しいプロセスとして指定された実行可能ファイルを起動します。下図のように
複数のリクエストを実行すると、リクエストごとにプロセスが生成されます。
- 3 -マルチウィンドウについて
INTARFRM連携機能では、複数の表示ファイル（DSP）をOPENすることで、マルチウィンドウ（複数画面）を使用することができます。
マルチウィンドウでは、プログラムで最初にOPENした画面を親画面、親画面のOPENからCLOSEまでの間にOPENした画面を子画面と
呼びます。
マルチウィンドウには、以下の制限があります。
－ 画面をOPEN後、次の画面をOPENする前に1度はOPEN済み画面に対するREAD要求を行わなければなりません。
－ 子画面のCLOSE前に親画面に対するREAD要求はできません。
－ 画面のCLOSE順序は、画面をOPENした順序の逆順序で行わなければなりません。
－ 画面に対するWRITEデータは、READ要求のタイミングで画面に反映されます。親画面のWRITE直後(READ要求前)に子画面
にアクセスした場合、子画面が閉じられた後にREAD要求するまで親画面は更新できません。
セションについて
INTARFRM 連携機能では、JavaクライアントとCOBOLプログラムとの間でセションを持ちます。Webブラウザから起動されたCOBOL
アプリケーションは、起動要求したWebブラウザと通信する表示ファイル（DSP）のOPENからCLOSEまで1つのセションを引き継ぎます。
マルチウィンドウを使用する場合には、親画面のOPENからCLOSEまでが1つのセションになります。
セションの接続には、以下の制限があります。
－ セション終了（表示ファイル（DSP）のCLOSE）後は、そのCOBOLアプリケーションからセションを接続することはできません。アプ
リケーションを終了してください。
－ 1つのCOBOLアプリケーションが複数のセションを接続することはできません。
－ 1つのWebブラウザから複数のセションを接続することはできません。
- 4 -［2］ 管理者機能
管理者機能とは、管理サービス配下で動作するCOBOLプログラムを管理する管理者向けの機能です。
管理者機能では、動作中のCOBOLプログラムの起動時刻やプログラム名、サーバのログを参照でき、またCOBOLプログラムを強制的に
終了させることもできます。
これにより、管理者は現在のセションやCOBOLプログラムの状態を把握できます。
管理者機能には、利用目的に応じて、以下の2つのインターフェースがあります。
・ Webブラウザ上で実行可能なWebインターフェース
・ サーバのコンソール上から実行するコマンドインターフェース
WebインターフェースはGUIで操作可能なため、操作性においてコマンドインターフェースより優れます。また、Webインターフェースでは、
COBOLランタイムシステムが出力する実行時メッセージを確認することができます。
［3］ プレビュー機能
プレビュー機能とは、画面入出力機能を使用して実行されたCOBOLプログラムから帳票印刷を行う場合に、その印刷イメージをWebブ
ラウザ上に表示する機能です。
利用者はWebブラウザ上で印刷イメージを確認した後で、必要に応じて印刷イメージをクライアント環境に保存したり、クライアント環境に
接続されているプリンタ装置に出力したりできます。

# 第3章 連携機能の使い方

第3章 連携機能の使い方
ここでは、INTARFRM 連携機能の使い方を説明します。


## 3.1 作業概要

3.1 作業概要
INTARFRM 連携機能では、Migration CJC for INTARFRM 連携機能管理サービス（以降、管理サービスといいます）が、COBOLプロ
グラムを起動し、セションを管理します。
管理サービスは、NetCOBOLのインストール時に「Migration CJC for INTARFRM 連携機能管理サービス」としてWindowsのサービスに
登録されます。
Javaクライアントから管理サービスにCOBOLプログラムの起動要求が通知されると、管理サービスはCOBOLプログラムを起動します。
OPEN文が実行されると、COBOLランタイムシステムによってセションを確立し、WRITE文、READ文によってJavaクライアントとデータをやり
取りすることができます。CLOSE文が実行されると、セションを閉じます。
この管理サービスを利用する為には、あらかじめ管理サービスの起動に必要な情報を設定しておく必要があります。
また、管理者は管理サービス配下で動作するCOBOLプログラムを、管理者機能を使用してWebブラウザ上から管理することができます。
また、コマンドプロンプトでも同様の操作を行うことができます。
管理者機能をWebブラウザ上から使用する場合、管理者機能をCGIプログラムとして動作するよう設定する必要があります。
注意
管理者機能を使用してWebブラウザ上からCOBOLプログラムを管理する場合には、管理者向けの機能が第三者から不正に利用される
危険性がないよう、CGIプログラムに対するユーザー承認およびアクセス制限を、必ず実施してください。


## 3.2 作業手順

3.2 作業手順
以下に、管理サービスを利用する際の作業手順を示します。
［1］ 管理サービスの動作環境設定
INTARFRM 連携機能を利用するためには、管理サービスの動作環境を設定する必要があります。
動作環境設定方法および設定内容については、“3.3 管理サービスの動作環境設定”を参照してください。
［2］ セションタイムアウト値の指定
JavaクライアントとCOBOLプログラムとのセションのタイムアウト値を設定します。
設定方法および設定内容は、“3.4 セションタイムアウト値の設定”を参照してください。
- 6 -［3］ 管理サービスの起動
管理サービスは、Windowsのサービスに「Migration CJC for INTARFRM 連携機能管理サービス」として登録されています。以下の手
順で起動してください。起動には、Administrator権限が必要です。
1. ［スタートメニュー］－［管理ツール］－［サービス］を実行します。
2. 一覧から「Migration CJC for INTARFRM 連携機能管理サービス」を選択します。
3. メニューの［サービスの開始］ボタンを押して起動します。
スタートアップの種別の初期設定値は「手動」です。必要に応じて「自動」に変更してください。
管理サービスが起動すると、JavaクライアントからCOBOLプログラムの起動要求を受け付け可能になります。
［4］ 管理者機能の利用
管理サービスの起動中は、管理者機能を実行してセション状態の確認・操作、サーバログの参照やログモードの変更を行うことがで
きます。
管理者機能の詳細は、“3.5 管理者機能”を参照してください。
［5］ 管理サービスの終了
管理サービスは、システムのシャットダウン時に終了します。また、Administratorを持つ管理者が手動で終了させることもできます。こ
のとき、継続中のセションの終了を待ってから終了するか、強制的に終了するかを管理サービスの動作環境であらかじめ設定して
おきます。
注意
継続中のセションの終了を待ってから終了する場合、全ての継続中のセションが一定時間内に終了しない時、管理サービスが正常に終了
できないことがあります。
システムのシャットダウンの場合には、管理サービスの動作環境の設定に関係なく、全ての継続中のセションを強制的に終了させます。


## 3.3 管理サービスの動作環境設定

3.3 管理サービスの動作環境設定
ここでは、管理サービスの動作環境の設定方法と設定内容を説明します。
管理サービスの動作環境の設定は、設定情報ファイルに記述します。設定情報ファイルは管理サービスの起動時に読み込まれます。


### 3.3.1 設定情報ファイル

3.3.1 設定情報ファイル


#### 3.3.1.1 設定情報ファイルの作成方法

3.3.1.1 設定情報ファイルの作成方法

## 3.4 セションタイムアウト値の設定

3.4 セションタイムアウト値の設定
INTARFRM 連携機能では、JavaクライアントとCOBOLプログラムとの間でセションを持ちます。このとき、互いに要求・応答の待ち状態が
発生しますが、正しく待ち状態が解消されないときのために、セションのタイムアウト値を設定できます。
セションに対して無応答のとき、セションのタイムアウト値を経過すると、セションは強制的に終了します。これにより、クライアントには例外が
通知され、COBOLプログラムでは、OPEN/READ/WRITE/CLOSE文がエラー復帰します。
例
利用者がブラウザを閉じたことに起因してタイムアウトが発生する例
セションタイムアウト値は、以下の環境変数で設定します。初期値は120秒です。
-----------------------------------------------------------------------------------------
@CBR_CJCSES_TIMEOUT = タイムアウト時間（秒）
-----------------------------------------------------------------------------------------
- 12 -ポイント
セションタイムアウト値は、セションを通じてメッセージのやり取りが行われると初期値にリセットされ、要求・応答の待ち状態が発生すると、
初期値からカウントダウンが始まります。
画面の応答に時間がかかったり、COBOLプログラムの処理に時間がかかったりする場合でも、セションタイムアウトは発生するため、運用
するシステムの適切な値を設定してください。

3.5.2.2 cobcjcsseコマンド
機能
開かれているセションを強制的に終了します。これにより、クライアントには例外が通知されます。また、COBOLプログラムでは、OPEN/

## 3.7 ログ

3.7 ログ
管理サービスは、設定情報ファイルの設定内容に基づいて、ログファイルを出力します。
ログの出力レベルには、エラーレベルと情報レベルがあります。
エラーレベル
管理サービスの検出したエラーの情報を出力します。サーバで発生したエラーがファイルに取得されます。
情報レベル
エラーレベルの内容に加えて、管理サービスの開始・終了と、セションの開始・終了などのCOBOLプログラムの動作に関する情報を出
力します。
サーバで発生したエラーがわかるように、サーバの動作中は情報レベル、またはエラーレベルを指定することをおすすめします。
ポイント
管理サービスは、ログファイルへの出力に加え、簡易的なメッセージをイベントログに出力します。出力するメッセージには、識別子として
[F4AGCJMG]を付加します。


### 3.7.1 ログファイルの出力形式

3.7.1 ログファイルの出力形式
ログの出力形式を以下に示します。
YYYY/MM/DD HH:MM:SS [Level] message
YYYY/MM/DD
メッセージを出力した年月日情報
HH:MM:SS
メッセージを出力した時刻情報
[Level]
メッセージの出力レベル
種別 意味
ERROR エラーレベルのメッセージを示します。
INFO 情報レベルのメッセージを示します。
- 21 -message
メッセージ内容が出力されます。


### 3.7.2 ログの仕様

3.7.2 ログの仕様
